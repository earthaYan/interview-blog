# tailwindcss

## 引入

- @tailwind base：引入 Tailwind 的基础样式（normalize.css 样式重置 + 基本全局样式）。
- @tailwind components：引入 Tailwind 的组件层（可复用 class 样式片段）。
- @tailwind utilities：引入 Tailwind 的工具类（比如 text-center、p-4 等）

::: code-group

```css [global.css-v3]
@tailwind base;
@tailwind components;
@tailwind utilities;
```

```css [global.css-v4]
@import "tailwindcss";
```

:::
类名组成：

- [属性]-[值]-[状态/媒体查询修饰符]：`bg-red-500`
- [修饰符:可选][属性]-[值]: `hover:w-64`

## tailwindcss 和 css module

tailwindcss 属于原子化 css 思想，所有页面共享一份 css 文件，但是类名会作用到单独的元素上。

css module 可以将 css 类默认限制在某个组件内，减少样式冲突的风险

## clsx 库

作用：快速切换类名

```js
import clsx from 'clsx';

export default function InvoiceStatus({ status }: { status: string }) {
  return (
    <span
      className={clsx(
        'inline-flex items-center rounded-full px-2 py-1 text-sm',
        {
          'bg-gray-100 text-gray-500': status === 'pending',
          'bg-green-500 text-white': status === 'paid',
        },
      )}
    >
    // ...
)}
```

::: tip
相对于传统的 classnames 库，clsx 更加轻量
:::

```js
// classnames
classNames("btn", { "btn-primary": isPrimary });
// clsx
clsx("btn", isPrimary && "btn-primary");
```

## postcss 配置

::: code-group

```js [v4]
const config = {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};

export default config;
```

```js [v3]
module.exports = {
  plugins: {
    tailwindcss: {},
  },
};
```

:::

tailwindcss 默认移动端优先，加前缀的类名只在 指定宽度及以上 生效

| 前缀   | 最小宽度 (min-width) |
| ------ | -------------------- |
| `sm:`  | 640px                |
| `md:`  | 768px                |
| `lg:`  | 1024px               |
| `xl:`  | 1280px               |
| `2xl:` | 1536px               |
