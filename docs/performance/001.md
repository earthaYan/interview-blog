# 核心 web 指标

![性能优化](.//imgs/better.png)

作用：用于量化用户真实体验，把主观感受转化为实际数据

## LCP

Largest Contentful Paint:用来衡量页面加载性能，页面开始加载到视口内**最大可见**图片或文本渲染完成的时间。需要`<=2.5s`

## CLS

Cumulative Layout Shift:衡量视觉稳定性，页面加载过程中元素的**非预期位置移动** ，需要`<0.1`。

开发时给图片等异步加载内容预留空间，保证位置相对稳定

## INP

Interaction to Next Paint:衡量整体交互响应性，交互到下次绘制时间，选取**最慢的一次交互**作为代表值

## 收集数据

### 实验室数据

- lighthouse：实验室数据，chrome 内置工具，支持三大指标评分
- [PageSpeed Insights](https://pagespeed.web.dev/analysis):结合实验室和现场数据，一站式性能报告

### 现场数据

- [Google Search Console](https://search.google.com/search-console)：提供网站整体核心指标健康报告
- web-vitals ：js 库，轻量收集三大指标数据上报并发送到自己的分析后台去深度分析

通过以上工具可以建立性能监测体系，识别修复用户性能瓶颈

## 其他指标

1. FP/FCP:首次绘制时间,用户从白屏到看到第一个内容的时长
2. TTFB：首字节时间，浏览器发起请求到接受响应的第一个字节时长，用来衡量网络和后端性能
3. Resource Timing:每个资源的加载时间和成功率
