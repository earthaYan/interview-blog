# 骨架屏 vs (Loading Spinner)

在等待异步数据的时候，通常页面会展示一个 Loading Spinner,不知道要等多久，也不知道是什么内容，导致用户体验非常糟糕。所以需要骨架屏来做一个优化

## 骨架屏

核心优势：提升感知性能，缓解用户焦虑

在数据加载完成前，用占位元素预先绘制出页面或组件的大致轮廓，通过模拟界面结构，提前告知即将加载的内容布局

|          | loading            | 骨架屏                     |
| -------- | ------------------ | -------------------------- |
| 用户感知 | 内容未知，焦虑     | 模拟内容轮廓，缓解焦虑     |
| 信息传达 | 正在加载，信息单一 | 预先展示页面布局，信息丰富 |
| 实现成本 | 简单               | 需要为特定页面定制         |
| 使用场景 | 局部异步操作       | 页面级、复杂模块的初始加载 |
