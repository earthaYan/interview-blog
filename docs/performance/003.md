# 资源提示符

资源提示是 link 标签 rel 属性的一组值，告诉浏览器未来即将发生的资源的处理策略，提前做准备。

## 作用

1. 提高网页的首屏加载性能
2. 减少网络连接中的各种延迟，比如 DNS,TCP,TLS
3. 预加载关键资源或预测性资源

## 属性

- 资源使用时机维度：当前页面用 vs 以后的页面用，
- 深度：解析域名->建立连接->下载资源

1. 提前解析域名：`<link rel="dns-prefetch" href="https://api.example.com"/>`，适合非关键的第三方资源，比如分析、监控脚本，可以作为 preconnect 的降级方案
2. 提前建立连接：`<link rel="preconnect" href="https://cdn.example.com"/>`,适合关键的第三方域名，比如 CDN 域名，API 域名,必须小于 6 个
3. 以最高优先级下载：`<link rel="preload" href="/fonts/main.woff2"" as="font" type="font/woff2" crossorigin/>`,下载后不执行，适合 css 定义的字体文件、背景图或者 LCP 图片，首屏必须的动态 js 脚本
4. 空闲时请求资源：`<link rel="prefetch" href="/js/next-page.js" as="script">`，适合 SPA 路由悬停的时候预取下一个 chunk，路由切换更快,
