# 兼容性

浏览器兼容性目前大部分 JS 兼容性问题可以通过 babel 和 polyfill 解决，比如 babel 解决了 ES6 的语法转换，polyfill 则解决了浏览器新 API 的兼容，css 兼容性问题主要通过添加浏览器内核前缀和优雅降级处理。

早期开发中遇到的移动端兼容性问题：

1. 设置样式`overflow:scroll/auto`，在 ios 系统上滑动会出现卡顿:通过设置`-webkit-overflow-scrolling:touch`解决
2. ios 下取消 input 输入框在输入英文时候首字母默认大写：`<input autocapitalize="off" autocorrect="on"/>`
3. ios 环境下 input 按钮设置了`disabled:true`显示异常：给`input[type=button][disabled] {opacity:1}`
4. android 环境下 input 设置`行高` placeholder 文字会偏上:input 有 placeholder 属性的时候不要设置`line-height`
5. 移动端字体小于 12px 的时候会显示为 12px：通过 transform 进行缩小
6. android 取消语音输入按钮：`input::-webkit-input-speech-button{display:none}`
7. 禁用 ios 和 android 用户选中文字：添加全局样式`{-webkit-user-select:none}`
8. 禁止 ios 弹出各种窗口：添加全局样式`{-webkit-touch-callout:none}`
